# Workspaces

Workspaces are the session-based or review-based product surfaces.

## Included Pages

- [`creative.md`](./creative.md)
- [`dreams.md`](./dreams.md)
- [`journal.md`](./journal.md)

## Shared Pattern

These workspaces all follow the same idea:

- create a draft session
- gather bounded context
- generate a first result
- persist pipeline events
- validate and evaluate
- repair once when needed
- save or publish only from a passing final state

That pattern keeps the work inspectable and makes the UI honest about what the server actually did.
