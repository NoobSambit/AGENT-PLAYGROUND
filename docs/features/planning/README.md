The planning section covers alternate futures, adversarial testing, and debate-style execution.

- [`scenarios.md`](./scenarios.md)

## Shared Pattern

These flows turn one of three things into inspectable output:

- a branch point from history
- a challenge template and participant set
- an arena topic and debate seat setup

They all need good stage logs, clear result summaries, and a way to show why a run ended the way it did.
